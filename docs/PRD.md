# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：廚師在日常工作或發想新菜色時，需要一個方便、有系統的地方來收集、整理與尋找各種食譜，避免紙本記錄容易遺失或難以搜尋的問題。
- **目標用戶**：廚師 (包含專業廚師、私廚、或是對烹飪有高度熱忱的使用者)
- **核心價值主張**：提供一個專為廚師設計的數位食譜資料庫，讓他們能快速記錄、分類、檢索靈感，並有效管理食材與步驟。

## 2. 功能需求

1. **新增食譜**
   - 作為 *廚師*，我希望 *能夠快速建立一篇新食譜*，以便 *記錄我剛獲得的靈感或測試好的配方*。

2. **列出材料與步驟**
   - 作為 *廚師*，我希望 *能清楚列出並觀看每道菜所需的材料比例和詳細的料理步驟*，以便 *在廚房實作時能精準參考且不易出錯*。

3. **選食材**
   - 作為 *廚師*，我希望 *能從系統內建或自建的食材庫中快速選取食材加入食譜*，以便 *標準化我的食譜設計並更容易管理庫存清單*。

4. **食譜分類與標籤**
   - 作為 *廚師*，我希望 *能為每道食譜加上分類（如：開胃菜、主餐、甜點）與自訂標籤*，以便 *依據菜系或功能有條理地整理我的專屬食譜庫*。

5. **搜尋與過濾食譜**
   - 作為 *廚師*，我希望 *能透過關鍵字、分類或特定食材條件來過濾與搜尋食譜*，以便 *當我手邊只有特定材料時，或是正在構思菜單時，能快速找到適合的料理*。

## 3. 非功能需求
- **技術限制**：使用核心 Web 技術建置，並採用 Flask + Jinja2 + SQLite 作為後端架構。
- **效能與安全考量**：
  - 系統需具備基本的輸入驗證，防止 XSS (跨站腳本) 或 SQL Injection (資料隱碼) 等常見攻擊。
  - 對於文字檢索與食譜清單的載入效能需維持流暢（頁面回應時間小於 1 秒）。

## 4. MVP 範圍 (最低可行性產品)

### Must Have (必須具備)
- 基礎食譜管理 (CRUD：新增、列表、查看、編輯、刪除)
- 在食譜中列出材料清單與詳細步驟設計
- 獨立的食材選擇器 (能夠預存或自由新增食材)

### Should Have (應該具備)
- 食譜的自訂標籤與分類功能
- 簡易的關鍵字與食材搜尋機制

### Nice to Have (有了更好)
- 食譜完成圖片的上傳與展示功能
- 將食譜匯出或列印為友善格式的功能
- 自動換算食材份量的計算機
